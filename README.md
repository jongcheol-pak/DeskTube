# DeskTube

유튜브 동영상을 **바탕화면 배경**으로 재생하는 Windows 11 앱 (WinUI 3, MSIX 패키지형).

바탕화면 아이콘 **뒤**(WorkerW)에 배경창을 배치해, 아이콘·작업 표시줄을 가리지 않고 유튜브 영상이 배경화면처럼 재생됩니다.

## 빠른 사용법 (요약)

사용자용 전체 안내서는 **[help.md](help.md)** 참고 — 화면별 설명·FAQ·개인정보 안내가 담겨 있습니다.

1. **앱 실행** — 메인 창이 열리고 트레이(시계 옆)에 아이콘이 상주합니다.
2. **주소 붙여넣기** — 홈 화면 입력란에 유튜브 주소(일반 영상·쇼츠·실시간 라이브)를 넣고 **재생**을 누릅니다.
3. **배경에서 재생** — 창을 닫아도 트레이에서 계속 재생됩니다. 완전 종료는 트레이 메뉴의 **종료**.
4. **플레이리스트** — 자주 듣는 영상을 목록으로 담아 셔플듣기·전체듣기로 끝없이 반복 재생합니다.

알아 두면 좋은 기본 동작:

- 여러 모니터에 동시 재생할 수 있고, **소리는 지정한 1개 모니터에서만** 납니다.
- 재생할 수 없는 영상(삭제·비공개·임베드 금지)은 자동으로 건너뜁니다.
- 유튜브 **프리미엄 계정으로 로그인**하면 배경 재생에도 광고가 없습니다.
- **전체 화면 앱·배터리 절약 모드·화면 잠금** 시 자동 일시정지해 자원을 아낍니다.
- Windows 시작 시 자동 실행과 앱 시작 후 자동 재생이 기본으로 켜져 있습니다(설정에서 변경).
- 데이터는 모두 내 PC에만 저장되며 외부로 전송하지 않습니다.

## 핵심 기능

- **배경 재생**: 유튜브 URL(watch / youtu.be / shorts / embed / live)을 입력하면 바탕화면 배경에서 재생 — 홈의 마지막 입력 URL은 저장되어 재실행 시 입력란에 그대로 표시, 재생 불가 영상(삭제·비공개·임베드 금지)은 자동으로 다음 항목으로 건너뛰고 전 항목이 재생 불가면 정지 후 안내. 곡 로드·전환의 로드 공백 동안 해당 곡 썸네일을 표시하고 재생이 시작되면 페이드아웃(검은 화면 대신 앨범아트 전환) (FR-1)
- **다중 모니터**: 재생할 모니터 다중 선택, 전체 모니터 동기 재생(마스터-미러 시각 보정), 소리는 지정한 1개 모니터에서만 (FR-3·4)
- **플레이리스트**: 최대 100개 리스트 × 1000개 항목 — 생성·이름변경·삭제, 항목 추가·삭제·드래그/우클릭 메뉴 정렬, 재생 중 변경 즉시 반영, 화면 진입 시 마지막에 선택한 리스트 기본 선택(없으면 첫 리스트), 지금 재생 중인 리스트는 목록과 홈 빠른 재생 칩에 코럴 스피커 아이콘으로 표시(일시정지 중에도 유지) (FR-6·18)
- **차트형 항목 목록**: 항목마다 순위·썸네일·제목·채널명·재생시간을 표시(제목·채널명은 유튜브 oEmbed 자동 조회·로컬 캐시, 실패 시 URL 표시 / 재생시간은 곡을 실제 재생할 때 수집·캐시하며 아직 재생하지 않은 곡은 공란), 셔플듣기·전체듣기 버튼과 행별 "이 곡부터 듣기" 재생, 우클릭 공유(URL 확인·클립보드 복사 팝업) 지원, 지금 재생 중인 곡은 순위 자리에 코럴 스피커 아이콘으로 표시(곡 전환 시 이동·일시정지 중 유지) (FR-18)
- **재생/정지 토글**: 선택한 리스트가 재생(일시정지 포함) 중이면 재생을 시작한 듣기 버튼(셔플듣기/전체듣기)의 문구 앞 아이콘이 정지 아이콘으로 바뀌어 어느 모드로 재생 중인지 구분되고, 그 버튼을 클릭하면 바로 정지. 재생 중인 곡의 행 버튼도 정지 버튼으로 바뀜(정지·곡 전환 시 원복) (FR-18)
- **반복 재생**: 목록이 끝나도 멈추지 않고 처음부터 계속 반복 — 재생 순서는 플레이리스트 화면에서 선택(셔플듣기 = 셔플 반복, 전체듣기·행 재생 = 순서대로 반복) (FR-7)
- **트레이 상주**: 창을 닫아도 트레이 메뉴(재생↔정지·음소거↔음소거 해제 상태 연동 토글, 설정 열기, 종료)로 제어, 더블클릭으로 설정 열기 — 트레이 재생도 마지막 항목부터 재개 (FR-9)
- **부팅 자동 시작**: Windows 로그인 시 창 없이 트레이로 시작 — "앱 시작 후 자동 재생"(FR-19)이 켜져 있으면 마지막 재생 설정으로 마지막 재생 항목부터 이어서 자동 재생하고, 꺼져 있으면 재생 없이 트레이만 상주. 신규 설치는 기본 켬 (FR-8)
- **중복 실행 방지(단일 인스턴스)**: 앱이 이미 실행 중일 때 다시 실행하면 새 창을 띄우지 않고 실행 중인 앱의 메인창을 표시·전면 활성화(트레이 상주 중에도 동일). 사용자가 먼저 실행해 둔 상태에서 부팅 자동 시작이 중복 도착하면 실행 중인 앱에 아무 영향 없이 무시(창 표시·자동 재생 없음) (FR-22)
- **앱 시작 후 자동 재생**: 설정 토글(기본 켜짐) — 켜면 앱이 시작될 때 마지막으로 재생하던 항목부터 배경 재생을 자동 시작. 직접 실행과 부팅 자동 시작 모두에 적용되며, 끄면 어느 경로로 시작해도 자동 재생하지 않음. 마지막 재생이 홈 즉시 재생("빠른 재생")이면 자동 재생하지 않음 (FR-19)
- **유튜브 로그인**: 본인 구글 계정으로 로그인하면 프리미엄 혜택(광고 없음)이 배경 재생에도 적용 — 홈 화면 우상단에 로그인 상태(로그인됨/안 됨)·구글 아이콘 로그인/로그아웃 버튼·프리미엄 안내 문구 표시, 세션 쿠키는 로컬 WebView2 프로필에만 저장 (FR-15)
- **자동 일시정지**: 전체 화면 앱 사용 중·배터리 절약 모드·화면 잠금 시 재생을 멈추고 플레이어를 절전 상태로 전환해 CPU·메모리 회수(절전 중 배경은 검게 표시, 해제 시 자동 재개 — 직접 누른 일시정지는 화면 유지) (NFR-1, 토글 3종)
- **화질 스케일**: 렌더링 세로 해상도 제한(원본/1440/1080/720/480)으로 시스템 부하 조절 (FR-13)
- **음소거 토글**: 설정의 음소거 카드 또는 홈 화면 모니터 카드의 소리 배지 클릭으로 켬/끔(배지 클릭은 홈 한정, 기본 음소거 켬) — 배지는 오디오 대상 모니터에 항상 표시되어 소리(코럴)/음소거(회색) 상태를 구분, 트레이 음소거 토글과 상태 공유·즉시 동기화 (FR-5)
- **동영상 크기 모드**: 채움(16:9로 화면 덮고 크롭) / 맞춤(영상비 유지 레터박스, 기본) / 늘리기(화면에 꽉 차게 왜곡) (FR-16)
- **자막 토글**: 설정의 자막 카드로 켬(강제 표시)/끔(강제 숨김, 기본) — 유튜브 계정의 자막 선호와 무관하게 앱이 제어, 재생 중 즉시 반영·곡 전환 후 유지 (FR-20)
- **다크 전용 디자인**: 코럴 포인트(#F25C54)의 다크 UI 고정 — Claude Design 시안(DeskTube 1a) 기반, 색·치수는 `Resources/DesignTokens.xaml` 토큰으로 관리 (테마 변경 기능 없음 — FR-17 폐기)
- **부하 절감 옵션**: 소리 없는(미러) 모니터 화질을 720 이하로 하향(기본 켬 — 설정에서 끔 가능, 기존 사용자의 선택 유지) + 정지·창 숨김 시 메모리 자동 반환 (NFR-2)
- **창 상태 복원**: 설정 창 크기·위치를 재시작 후 복원(WinUIEx), 커스텀 타이틀바(44px, 시스템 캡션 버튼)
- **다국어**: 한국어/English + 시스템 언어 추종, 앱 이름도 현지화(한국어 환경은 시작메뉴·창 제목·정보 화면·트레이까지 "데스크튜브") (NFR-4)
- **정보 화면**: 버전(카드 오른쪽 끝 공식 홈페이지 링크)·개인정보 처리방침 요지·개발 지원(후원) 링크(클릭 시 GitHub Sponsors로 이동)·오픈소스 라이선스 목록(항목 클릭 시 해당 라이브러리 공식 사이트로 이동, 전문 파일은 패키지 동봉) (FR-11·12·21)

## 실행 방법

### 개발 실행
Visual Studio 2026에서 `DeskTube.slnx`를 열고 x64 + `DeskTube (Package)` 프로필로 F5 (패키지형이라 Package 프로필 필수).

### 빌드·테스트 (CLI)
```
dotnet build DeskTube.slnx -c Debug -p:Platform=x64
dotnet test tests/DeskTube.Tests/DeskTube.Tests.csproj -c Debug -p:Platform=x64
dotnet format DeskTube.slnx --verify-no-changes
```

### MSIX 패키징 (사이드로드)
```
MSBuild.exe src/DeskTube/DeskTube.csproj -restore -p:Configuration=Release -p:Platform=x64 ^
  -p:GenerateAppxPackageOnBuild=true -p:UapAppxPackageBuildMode=SideloadOnly -p:AppxBundle=Never
```
산출물: `AppPackages/DeskTube_<버전>_x64_Test/DeskTube_<버전>_x64.msix` (개발용 미서명 — Store 제출 시 Store가 서명)

## 아키텍처

계층형(서비스 중심) + MVVM(CommunityToolkit.Mvvm). 도메인 레이어 분리 없음 — 규칙이 얇은 앱에 과한 추상화를 피함.

```
src/DeskTube/
├── App.xaml.cs            # 진입점 — 언어 선적용, 조용 시작 분기, 컴포지션 루트(AppServices) 초기화
├── MainWindow.xaml        # NavigationView 셸 (홈/플레이리스트/정보 + 설정)
├── Views/                 # HomePage(URL 즉시 재생), PlaylistsPage, SettingsPage, AboutPage,
│                          # LoginWindow(유튜브 로그인)
├── ViewModels/            # 페이지별 VM — App.Services(수동 컴포지션 루트) 소비
├── Services/
│   ├── PlaybackCoordinator# 재생 상태 단일 소유자 — 큐 진행·다중 모니터 동기·오디오 라우팅·크기 모드·미러 하향
│   ├── WallpaperHost      # WorkerW 부착/해제 (24H2 자식·구형 형제 이중 탐색), 배경 복구
│   ├── PlayerHost         # CoreWebView2Controller + player.html IFrame 브리지 (가상 호스트 서빙)
│   ├── PlaylistLibrary    # 리스트 CRUD + 상한 (100×1000)
│   ├── MonitorService     # 모니터 열거·변경 감지(WM_DISPLAYCHANGE)·대상 해석
│   ├── PowerPolicyService # 자동 일시정지 신호 합성 (전체화면 QUNS·배터리 세이버·세션 잠금)
│   ├── TrayIconService    # H.NotifyIcon 트레이 (상태 연동 토글 2항목 + 설정/종료, PopupMenu 모드)
│   ├── StartupService     # StartupTask 상태·조용 시작 판별 (-startup 인자 폴백)
│   ├── YouTubeSessionService # 로그인 세션 (SAPISID 쿠키) 확인·로그아웃
│   └── JsonStateStore     # settings/playlists JSON 원자적 영속화 (손상 .bak 복구)
├── Interop/               # WallpaperSurface(Win32 배경창)·WorkerW·모니터·세션·워킹셋 P/Invoke
└── Strings/               # en-US(폴백)·ko-KR .resw
```

### 재생 플로우
1. URL 입력(홈) 또는 플레이리스트 재생 → `PlaybackCoordinator.StartAsync`
2. 선택 모니터마다 `WallpaperHost.Attach`(WorkerW 뒤 배치) + `PlayerHost` 생성(WebView2)
3. `player.html`의 유튜브 IFrame API가 재생 — 오디오 대상 1곳만 소리, 나머지 음소거+시각 동기
4. 정지 시 전부 해제하고 원래 배경화면 복구

### 데이터
- 설정·플레이리스트: `ApplicationData.Current.LocalFolder`의 `settings.json` / `playlists.json` (원자적 쓰기, 손상 시 .bak 보존 후 기본값)
- 로그인 쿠키: 동일 폴더의 WebView2 프로필 (앱은 계정 정보를 저장하지 않음 — [개인정보 처리방침](docs/privacy-policy.md))

## Store 제출 (사용자 수행)

1. 현재 identity는 임시(`DeskTube.Dev`) — 파트너 센터에서 발급받은 identity로 `Package.appxmanifest` 교체 후 재패키징
