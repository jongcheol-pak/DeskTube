## Phase 3 — Root Cause (검증된 메커니즘)
1. **앱은 이 영상을 전 기간 정상 재생** — 오류 0, 진행 확인, 현재는 디코드·렌더·배경 표시까지 실측 정상.
2. **소리가 안 난 이유**: ① 앱 음소거(`IsMuted: true`) ② 이 MV는 도입부 ~11초가 사실상 무음.
3. **화면이 검정인 이유**: ① 사용자 바탕화면이 원래 검정(이미지 없음·배경색 0 0 0)이라 정지·로드 초기·MV 어두운 도입부가 전부 "검정 = 미표시"로 보임 ② 재시도 간격 4~10초 → 매번 0초부터 재시작해 밝은 장면·음악 구간(11초~)에 도달하기 전에 재시작 반복.
4. **워크어라운드("웹에서 재생 후 되면 됨")의 실체(추정)**: 웹에서 곡이 정상임을 확인한 뒤에는 앱에서 도입부를 넘겨 기다리게 되어 표시·소리를 목격 — 대기 시간 차이가 핵심일 개연성이 높음(계정 상태 인과는 배제 못 하나, 현재 앱 환경 실측으로 파이프라인 자체는 결함 없음 확정).

## Phase 4 — Fix

  - **음소거 해제**(현재 `IsMuted: true`) 후 볼륨 확인.

- **재발 시 판정 절차**: 검정 상태에서 재시작하지 말고 15초 대기 → 여전히 검정+무음이면 그 시각을 기록하고 로그의 `start-watchdog` 진단과 대조. (H5 검증용 — 필요 시 player.html 진단에 프레임 카운트 추가 검토.)
