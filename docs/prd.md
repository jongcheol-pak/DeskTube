# PRD: DeskTube — 유튜브 동영상 배경화면 앱

> 상태: **승인됨 (2026-07-15)** — 이 문서는 고정. 변경은 사용자 합의 필수.

## 목적

윈도우 사용자가 유튜브 동영상을 바탕화면 배경(아이콘 뒤)으로 재생할 수 있게 하는 WinUI 3 데스크톱 앱.
Microsoft Store 배포를 목표로 하며, 멀티 모니터 환경과 저부하(백그라운드 상시 실행) 동작을 지원한다.

## 플랫폼·기술 제약 (질문 라운드 확정)

- **지원 OS**: Windows 11 전용 (Win10은 2025-10 일반 지원 종료)
- **재생 엔진**: WebView2 + 유튜브 공식 IFrame Player API (ToS 준수, Store 심사 안전). 스트림 추출 방식 금지.
- **임베드 도메인**: `www.youtube.com/embed` 사용 (로그인 쿠키 적용을 위해 `youtube-nocookie.com` 사용 불가)
- **화질**: IFrame API의 `setPlaybackQuality`는 no-op — 화질 강제 불가. 간접 제어(렌더 해상도 스케일)만 제공.
- **배포**: MSIX 패키징 + Microsoft Store, WACK 통과 필수

## 기능 요구사항 (Functional Requirements)

| ID | 요구사항 | 우선순위 | 검증 방법 |
|----|---------|---------|----------|
| FR-1 | 유튜브 URL(watch/youtu.be/shorts 형식)을 입력하면 영상 ID를 파싱·검증하고 배경화면으로 재생한다 (WebView2 + 공식 IFrame Player API) | Must | 단위테스트: URL 파싱(정상/비정상 케이스) + 재생은 HUMAN-VERIFY |
| FR-2 | 재생 창은 바탕화면 아이콘 뒤(WorkerW 레이어)에 배치되어 아이콘·창을 가리지 않는다. Windows 11 24H2 전후의 WorkerW 구조 차이를 모두 지원한다 | Must | HUMAN-VERIFY: 아이콘 표시 상태에서 영상이 아이콘 뒤에 렌더링 |
| FR-3 | 연결된 모니터를 열거해 1개 또는 다중 모니터를 배경화면 대상으로 선택할 수 있다. 다중 선택 시 모든 대상 모니터에 동일 영상을 동기 재생한다 | Must | 단위테스트: 모니터 열거·선택 저장/복원 + 표시는 HUMAN-VERIFY |
| FR-4 | 다중 모니터 재생 시에도 오디오는 사용자가 지정한 1개 모니터의 플레이어에서만 출력된다 (기본값: 주 모니터, 나머지 강제 음소거) | Must | 단위테스트: 오디오 대상 결정 로직 + 청취는 HUMAN-VERIFY |
| FR-5 | 볼륨(0~100)과 음소거를 설정할 수 있고 재시작 후에도 유지된다 | Must | 단위테스트: 볼륨 상태 저장/복원 |
| FR-6 | 플레이리스트를 생성/이름변경/삭제하고 항목(유튜브 URL)을 추가/삭제/순서변경할 수 있다. 리스트 최대 100개, 리스트당 항목 최대 1000개 제한을 강제한다 | Must | 단위테스트: CRUD + 상한 초과 시 거부 |
| FR-7 | 재생 모드를 지원한다 — 순차 / 셔플(전곡 무작위 1회 순회) / 랜덤(중복 허용 무작위) / 반복(현재 항목·전체 목록) | Must | 단위테스트: 각 모드의 다음 곡 결정 로직 |
| FR-8 | 윈도우 부팅 시 자동 실행(MSIX StartupTask)을 켜고 끌 수 있으며, 자동 실행 시 설정 창 없이 트레이 아이콘 상태로 시작하고 마지막 재생 설정(재생목록·모니터·볼륨)으로 배경 재생을 자동 시작한다 | Must | 단위테스트: 시작 인자 판별·상태 복원 로직 + StartupTask 등록·자동 재생은 HUMAN-VERIFY |
| FR-9 | 트레이 아이콘을 상시 표시하고 컨텍스트 메뉴로 재생, 정지, 볼륨 on/off, 종료를 제어할 수 있다. 설정 창을 닫아도 앱은 트레이에서 계속 동작한다 | Must | HUMAN-VERIFY: 트레이 메뉴 4개 동작 |
| FR-10 | 설정 화면에서 모니터 선택·URL 입력·플레이리스트 관리·볼륨·오디오 출력 모니터·재생 모드·자동 실행·화질(간접)을 설정할 수 있다 | Must | HUMAN-VERIFY: 설정 화면 각 항목 동작 |
| FR-11 | 앱 정보 화면(앱 이름·버전·개발자)을 표시한다 | Must | HUMAN-VERIFY |
| FR-12 | 오픈소스 라이선스 화면(사용한 라이브러리와 라이선스 전문)을 표시한다 | Must | HUMAN-VERIFY + 사용 패키지 목록과 화면 목록 일치 확인 |
| FR-13 | 화질 간접 제어: 플레이어 렌더 해상도 스케일(예: 720p/1080p/원본)을 설정할 수 있고, 설정 화면에 "최종 화질은 유튜브가 자동 결정" 안내 문구를 표시한다 | Must | 단위테스트: 스케일 → 임베드 크기 계산 로직 + 안내 문구는 HUMAN-VERIFY |
| FR-14 | 플레이리스트·설정은 로컬(ApplicationData)에 영속화되어 재시작 후 복원된다 | Must | 단위테스트: 저장 → 재로드 왕복 일치 |
| FR-15 | 유튜브 계정 로그인 창(WebView2로 youtube.com 로그인)을 제공하고 세션(쿠키)을 배경 플레이어와 공유해, 프리미엄 사용자는 광고 없이 재생된다. 로그아웃 기능 포함. 구글의 임베디드 브라우저 로그인 차단(외부 변수) 시에는 로그인 실패를 안내하고 앱의 나머지 기능은 정상 동작한다 | Should | HUMAN-VERIFY: 로그인 → 임베드에 세션 적용 확인 (광고 미표시는 프리미엄 계정 + 구글 정책에 의존) |

## 비기능 요구사항 (Non-Functional)

| ID | 요구사항 | 검증 방법 |
|----|---------|----------|
| NFR-1 | 자동 일시정지: ① 다른 앱 전체화면 시 ② 배터리 세이버 시 ③ 화면 잠금·세션 잠금 시 배경 재생을 일시정지하고, 해제되면 자동 재개한다 | 단위테스트: 상태 이벤트 → 일시정지/재개 결정 로직 + 실동작 HUMAN-VERIFY |
| NFR-2 | 메모리: 재생 모니터당 WebView2 인스턴스 1개만 유지(단일 프로필 공유로 브라우저 프로세스 공유), 재생 중지 시 WebView2 리소스 해제. 트레이 대기(재생 중지) 상태 워킹셋 목표 150MB 이하 | 작업 관리자 측정 (측정치 기록) |
| NFR-3 | 설정 창 콜드 스타트 3초 이내 | 수동 측정 |
| NFR-4 | UI 다국어: 한국어 + 영어 (resw 리소스 기반, OS 언어 자동 감지) | 언어 전환 후 문구 확인 |
| NFR-5 | Microsoft Store 제출 요건 충족: MSIX 패키징 + WACK(Windows App Certification Kit) 통과 | WACK 실행 결과 |
| NFR-6 | 개인정보: 로그인 쿠키는 앱 로컬 WebView2 프로필에만 저장, 외부 전송 없음. Store 제출용 개인정보처리방침 문서 포함 | 문서 존재 + 코드에서 외부 전송 부재 확인 |

## Out of Scope (명시적 제외)

- 로컬 동영상 파일·유튜브 외 사이트 재생
- 유튜브 검색·탐색 브라우저 기능 (URL 직접 입력만)
- 비프리미엄 계정의 광고 제거·차단 (ToS 위반)
- 클라우드 동기화, 플레이리스트 공유
- 스트림 추출(yt-dlp 등) 방식 재생 — YouTube ToS 위반, Store 리젝 위험
- 모니터별 독립 플레이리스트 (전 모니터 동일 영상만 — Q4 확정)
- Windows 10 지원 (Q6 확정)

## 성공 기준

- active Must FR 100% 충족 (Phase G 재검증 기준)
- WACK 통과 (Store 제출 가능 상태)
- Windows 11 24H2 이후에서 배경 재생 정상 동작 (HUMAN-VERIFY)
